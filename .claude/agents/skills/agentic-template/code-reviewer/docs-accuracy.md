# Documentation Accuracy

- **Accuracy vs code** — Verify that documented APIs, parameters, return types, and behaviors match what the code actually does. Check for stale descriptions that refer to old interfaces or removed features.
- **Example validity** — Check that code examples use current API signatures and would compile/run correctly. Verify that example outputs match what the code would actually produce.
- **Completeness** — Check that all new features, configuration options, behavioral changes, and breaking changes are documented. Verify that deprecated functionality is marked.
- **Terminology consistency** — Verify that documentation uses the same terms as the code and plan. Check for synonyms that create confusion (e.g., "user" vs "account" vs "profile" for the same concept).
- **Cross-reference integrity** — Check that links, references, and "see also" pointers target existing sections or files. Verify that section names referenced from the plan or code match actual documentation headings.
